# Pages Definition (v2)

## 页面
1. `/login`
- 职责：账号密码登录；根据角色跳转用户端或管理端。

2. `/register`
- 职责：注册新用户，提交后进入待审批页。

3. `/pending-approval`
- 职责：展示“账号待管理员审批”状态。

4. `/home`
- 职责：个人主页首页，承载站点定位、最近文章、精选项目与技术快照。

5. `/blog`
- 职责：技术博客列表页，提供标签筛选与文章检索入口。

6. `/blog/:slug`
- 职责：技术博客详情页，提供高可读正文、目录和上一篇/下一篇导航。

7. `/projects`
- 职责：项目列表页，展示项目卡片、技术栈与状态。

8. `/projects/:id`
- 职责：项目详情页，展示问题背景、技术方案、结果复盘和相关链接。

9. `/admin`
- 职责：管理员后台，当前支持“待审批用户列表 + 同意注册”，并预留内容管理与站点设置入口。

## 全局导航约束
1. 登录后页面统一包含顶栏主导航：`Home / Blog / Projects`。
2. 内容页统一提供面包屑：`首页 / 模块 / 列表或详情`。
3. 面包屑应反映用户当前信息层级，不得与主标题语义重复。

## 边界
- 页面负责路由与数据编排。
- 组件负责可复用 UI（导航、面包屑、卡片容器等）。
- 管理端与用户端使用同一登录入口，按角色分流。
