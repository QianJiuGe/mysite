# Global Acceptance Criteria

## 用途
定义跨仓一致的验收底线。

## 通用验收项
1. 规格与实现一致：若接口或行为变更，spec 同步更新。
2. 测试可执行：至少包含关键路径与失败路径。
3. 文档可追踪：关键决策进入 implementation notes。
4. 风险可说明：列出影响面与回滚点。
5. 禁止未验证上线结论。

## v1 登录注册首页附加验收
1. 新注册用户默认 `pending`，审批前禁止登录。
2. 管理员审批后用户可登录。
3. 系统存在默认管理员账号（仅初始化场景）。
4. 首页支持登录用户进入，结构可扩展（不含 tab）。
