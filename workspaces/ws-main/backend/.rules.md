# Backend Rules

1. API 契约以 `spec/api/openapi.yaml`（后续 proto）为准。
2. Kratos 分层遵循 `server -> service -> biz -> data`。
3. 业务规则只在 `biz` 层定义。
4. 数据访问只在 `data` 层实现。
5. 所有新增行为必须有可执行测试场景。
6. 关键路径错误必须有明确错误码和语义。
7. 配置必须集中于 `configs/`，禁止散落常量。
8. 变更必须包含影响面与回滚点说明。
9. 禁止为未来假设提前实现复杂抽象。
10. 破坏性变更必须先写迁移说明。
11. 一次变更只解决一个主要问题。
12. 文档与代码不一致时，以更新文档为先。
