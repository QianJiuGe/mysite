# knowledge

## 仓库定位
全局知识库：沉淀跨前后端共享的规则、架构决策、产品方向与执行手册。

## 当前阶段目标
- 建立五阶段流程的最小可执行文档骨架。
- 提供 AI 可直接遵循的强约束入口。
- 避免文档膨胀，保证每个主题单一主文件。

## 目录说明
- `ai/`: AI 协作规则与技能清单。
- `product/`: roadmap 与 PRD 模板。
- `architecture/`: 系统总览与 ADR 模板。
- `conventions/`: 代码、UI、API 约定。
- `spec/`: 全局验收标准。
- `playbooks/`: feature/API/内容/发布/维护工作手册。
- `anti-patterns/`: 全局反模式与复盘留痕。

## 下一步建议
1. 先补 `product/prd.md` 的 v1 目标范围。
2. 再按 PRD 更新后端/前端 `spec`。
3. 每轮迭代至少更新一条规则或反模式。
