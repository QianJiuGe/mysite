# PRD: v1 登录注册 + 首页骨架

## 1. 问题定义
- 当前问题：网站尚无用户身份体系与首页承载能力，无法开始用户端/管理端迭代。
- 目标用户：访客用户、新注册用户、唯一管理员。
- 用户价值：用户可注册并等待审批，审批后可登录；管理员可完成审批；站点有可扩展首页入口。

## 2. 目标与非目标
- 目标（本期）
  1. 建立注册、登录、管理员审批闭环。
  2. 建立用户端首页（Q空间风格骨架，不做复杂模块）。
  3. 落地 MySQL + Redis 本地开发运行基础。
- 非目标（本期不做）
  1. 不做第三方登录。
  2. 不做多管理员与权限细分。
  3. 不做首页 tab 体系与复杂 feed 算法。

## 3. 范围
- In Scope
  - 用户注册（pending）
  - 管理员审批
  - 登录（仅 approved 用户）
  - 首页骨架渲染
  - Docker 启动 MySQL + Redis
- Out of Scope
  - 邮件/短信验证码
  - 密码找回
  - 社交关系与动态发布

## 4. 关键约束
- 技术约束：前端 Vue 3，后端 Go + Gin，存储 MySQL + Redis。
- 业务约束：管理员只有一个；新用户必须审批后可登录；管理员首登使用默认账号密码。
- 时间约束：先完成最小闭环可演进版本。

## 5. 验收标准
- 场景 1（注册待审批）
  - Given 新用户提交合法注册信息
  - When 调用注册接口成功
  - Then 用户状态为 `pending` 且不可直接登录
- 场景 2（管理员审批）
  - Given 存在 `pending` 用户
  - When 管理员执行审批
  - Then 用户状态变为 `approved`
- 场景 3（登录与首页）
  - Given 用户状态为 `approved`
  - When 登录成功并访问首页
  - Then 返回会话令牌并渲染首页骨架

## 6. 风险与后续
- 已知风险：默认管理员凭据若不尽快改密存在安全风险。
- Follow-up：补充首登强制改密、审计日志、限流与验证码。
