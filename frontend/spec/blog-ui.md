# Blog UI Spec (v2)

## 目标
提供可长期维护的技术博客阅读体验，强调信息密度与可读性平衡。

## 页面结构
1. 列表页（`/blog`）
- 顶部：面包屑 + 标题 + 简介。
- 筛选区：标签、关键词、年份（本阶段可先做标签与关键词占位）。
- 列表区：文章卡片，包含标题、摘要、标签、发布时间、阅读时长。

2. 详情页（`/blog/:slug`）
- 顶部：面包屑 + 文章标题 + 元信息（发布日期、更新日期、阅读时长）。
- 主体：正文排版容器。
- 侧边：目录导航（本阶段支持静态目录占位）。
- 底部：上一篇/下一篇导航。

## 视觉与排版约束
1. 正文字号和行高优先可读性：避免过小字号与紧凑行距。
2. 段落、代码块、引用块、列表之间必须有明确节奏间距。
3. 标题层级可通过字号和字重区分，不依赖颜色堆叠。
4. 元信息使用次级色，不得喧宾夺主。

## 本期范围
- 交付博客列表与详情结构、卡片样式、阅读容器和面包屑。
- 目录与筛选先采用前端静态数据，不阻塞 UI 结构落地。

## 本期不做
- 不接入 CMS 或 Markdown 渲染管线。
- 不实现全文检索后端。
- 不实现评论系统。
